# 肺炎CT影像识别系统 — 设计文档

## 一、项目概述

### 1.1 任务描述
利用卷积神经网络（CNN）对肺部 CT 影像进行分析，判断是否存在肺炎感染。
属于典型的医学图像二分类任务（阳性/阴性）。

### 1.2 推荐技术与数据集
- **框架**：PyTorch
- **模型**：ResNet（预训练迁移学习）
- **数据集**：COVID-CT Dataset

### 1.3 项目结构
```
pneumonia_ct_classifier/
├── config.py          # 所有超参数配置
├── dataset.py         # 数据集加载与预处理
├── model.py           # ResNet模型构建
├── train.py           # 训练主程序
├── evaluate.py        # 模型评估与指标计算
├── predict.py         # 单张图像推理预测
├── predict_api.py     # 预测 API 模块
├── utils.py           # 工具函数（可视化、日志等）
├── gradcam.py         # Grad-CAM 热力图可视化

├── requirements.txt   # 依赖清单

```

## 二、数据集分析

### 2.1 COVID-CT 数据集
该数据集由加州大学圣地亚哥分校收集，包含：
- **阳性样本**：COVID-19 患者 CT 图像
- **阴性样本**：非 COVID 患者 CT 图像

### 2.2 数据集获取方式
数据集来源：https://github.com/UCSD-AI4H/COVID-CT

该 GitHub 仓库提供了：
- 训练集、验证集、测试集的图像文件路径列表（txt文件）
- 所有 CT 切片图像

**实际下载方式**：
1. 访问 https://github.com/UCSD-AI4H/COVID-CT 下载数据
2. 或使用 Kaggle 镜像：https://www.kaggle.com/datasets/tawsifurrahman/covid19-ct

### 2.3 数据分布（训练前必须检查）
医学图像数据集常见严重类别不平衡，如阳性样本远多于阴性样本。
解决方案：**加权交叉熵损失函数（Weighted Cross Entropy Loss）**，自动根据各类别样本数量计算权重。

## 三、技术方案详细设计

### 3.1 为什么选择 ResNet50 预训练迁移学习？

| 方案 | 优点 | 缺点 | 选择理由 |
|------|------|------|----------|
| 从头训练 ResNet | 简单 | CT 图像与自然图像差异大，需要大量数据 | 医学数据集通常较小，从头训练容易过拟合 |
| ResNet18/34 | 轻量、速度快 | 特征提取能力较弱 | 对于医学图像细微特征可能不够 |
| **ResNet50 预训练** | 特征提取能力强，收敛快，泛化好 | 参数量较大 | **最优选：ImageNet 预训练权重提供强大的通用视觉特征** |
| EfficientNet | 精度更高 | 推理速度较慢，依赖复杂 | 对于期末作业来说过复杂 |

**核心原理**：ImageNet 预训练的 ResNet50 已经学会了提取边缘、纹理、形状等通用视觉特征。
迁移到 CT 图像时，底层特征（边缘检测等）可以直接复用，只需微调高层语义层。

### 3.2 图像预处理策略

#### 3.2.1 为什么要做以下处理？

```
输入 CT 图像 (灰度/PNG)
    │
    ├── ① 转 RGB 3通道  ← ResNet 输入必须是 3 通道
    │     原因：预训练 ResNet 权重是在 3 通道 RGB 图像上训练的
    │
    ├── ② 缩放到 224×224  ← ResNet 标准输入尺寸
    │     原因：预训练模型的全连接层输入尺寸固定，224×224 是标准选择
    │     过大会增加显存/计算，过小会丢失细节
    │
    ├── ③ 归一化 mean=[0.485,0.456,0.406], std=[0.229,0.224,0.225]
    │     原因：使用 ImageNet 统计的均值和标准差，与预训练权重的分布一致
    │     如果不一致，迁移学习效果会大幅下降
    │
    ├── ④ 随机水平翻转 + 随机旋转  ← 数据增强
    │     原因：医学图像中 CT 的左右翻转不影响诊断，增加样本多样性
    │     防止模型记住训练集的特定方位，提升泛化能力
    │
    └── ⑤ 仅 Resize（测试集不做增强）
          原因：测试时保持确定性，确保评估结果可复现
```

### 3.3 模型架构设计

```
预训练 ResNet50 (ImageNet权重)
    │
    ├── 冻结前两层 stage1-stage2 的参数  ← 保留通用特征提取能力
    │     原因：底层特征（边缘、纹理）是通用的，不需要重新学习
    │     减少可训练参数，降低过拟合风险
    │
    ├── 替换最后的分类层
    │     原: fc(in_features=2048, out_features=1000)  ← ImageNet的1000类
    │     新: fc(in_features=2048, out_features=2)     ← 肺炎二分类
    │
    └── 添加 Dropout(p=0.5) 在全连接层前
          原因：进一步防止过拟合，随机丢弃部分神经元
          对于小数据集的医学图像分类尤其重要
```

### 3.4 训练策略设计

#### 3.4.1 损失函数：加权交叉熵
```
为什么不用普通 BCE/CrossEntropy？
- 医学数据集通常类别不平衡（如 349 阳性 vs 397 阴性 或其他比例）
- 普通损失函数会偏向多数类，导致少数类识别率低
- 加权交叉熵 = -Σ(w_c * y_c * log(p_c))
  其中 w_c = 总样本数 / (类别数 * 类别c样本数)
  这样每个类别的贡献被归一化，模型不会偏向某一类
```

#### 3.4.2 优化器：AdamW
```
为什么不用 SGD？
- AdamW 自带自适应学习率，收敛更快
- 带权重衰减（Weight Decay），天然正则化
- 对于迁移学习任务，AdamW 通常比 SGD 表现更好
- 学习率设为 1e-4，足够小以免破坏预训练权重

为什么不用普通 Adam？
- AdamW 修正了 Adam 的权重衰减实现，泛化性能更好
```

#### 3.4.3 学习率调度：余弦退火
```
Cosine Annealing LR Scheduler
- 学习率从 1e-4 平滑衰减到 1e-6
- 余弦曲线衰减比阶梯式更平滑，不易陷入局部最优
- T_max = 总 epoch 数，每个 epoch 后自动更新
```

#### 3.4.4 早停策略（Early Stopping）
```
监控验证集 Loss：
- 如果连续 patience=15 个 epoch 验证集 loss 没有下降 → 停止训练
- 原因：防止过拟合，在模型开始记住训练集之前停止
- 同时保存验证集上表现最好的模型权重
```

#### 3.4.5 训练轮数
```
建议 epoch = 50-80
- 太少：模型还没充分学习
- 太多：过拟合风险增加
- 有 Early Stopping 保护，设大一点没关系
```

### 3.5 评估指标设计

| 指标 | 公式 | 为什么需要 |
|------|------|-----------|
| **Accuracy** | (TP+TN)/(TP+TN+FP+FN) | 总体正确率，但类别不平衡时不够可靠 |
| **Precision** | TP/(TP+FP) | 预测为阳性的样本中有多少是真的，避免误诊 |
| **Recall** | TP/(TP+FN) | 真实阳性中有多少被检出，避免漏诊（医学中极其重要） |
| **F1-Score** | 2*P*R/(P+R) | Precision 和 Recall 的调和平均，综合指标 |
| **AUC-ROC** | ROC 曲线下面积 | 不受分类阈值影响，评估模型整体排序能力 |
| **Confusion Matrix** | TP/FP/FN/TN 矩阵 | 直观展示模型在各类别上的表现 |

**特别注意**：医学诊断中 **Recall（召回率）比 Precision 更重要**，因为漏诊（假阴性）的代价远大于误诊（假阳性）。

## 四、完整实现步骤

### 步骤 1：环境准备
```bash
# 安装依赖
pip install torch torchvision numpy matplotlib scikit-learn seaborn opencv-python
```

### 步骤 2：数据集准备
1. 从 GitHub 或 Kaggle 下载 COVID-CT 数据集
2. 确保数据集路径正确，修改 `config.py` 中的 `DATA_DIR`

### 步骤 3：运行训练
```bash
python train.py
```
- 自动创建 results/ 目录
- 每个 epoch 输出训练/验证 loss 和 accuracy
- 自动保存最佳模型到 `results/checkpoints/best_model.pth`
- 训练结束后生成 loss 曲线和 accuracy 曲线

### 步骤 4：模型评估
```bash
python evaluate.py
```
- 在测试集上计算所有评估指标
- 输出混淆矩阵、ROC 曲线、分类报告
- 图表保存到 `results/figures/`

### 步骤 5：单张图像预测
```bash
python predict.py --image path/to/ct_image.png
```
- 输入一张 CT 图像
- 输出预测结果（阳性/阴性）和置信度
- 可视化输入图像及预测结果

### 步骤 6：Web 可视化界面
```bash
python app.py
```
- 启动 Web 界面，在浏览器打开 http://localhost:7860
- 功能包括：
  - 单张/批量预测
  - Grad-CAM 热力图可视化
  - 图像增强（亮度、对比度、锐度、平滑度）
  - 预设方案快速优化
  - 对比视图
  - PDF/图片报告导出
  - 训练曲线、ROC 曲线、混淆矩阵展示
  - 历史记录管理

## 五、关键技术决策总结

| 决策 | 选择 | 理由 |
|------|------|------|
| 模型 | ResNet50 预训练 | 特征提取能力强，迁移学习效果好 |
| 损失函数 | 加权交叉熵 | 处理类别不平衡 |
| 优化器 | AdamW (lr=1e-4) | 收敛快，自带正则化 |
| 学习率调度 | 余弦退火 | 平滑衰减，避免局部最优 |
| 数据增强 | 随机翻转+旋转 | 增加样本多样性，防止过拟合 |
| 正则化 | Dropout(0.5) + Weight Decay | 双重防过拟合 |
| 早停 | patience=15 | 自动找到最佳训练点 |
| 评估 | AUC + F1 + Recall | 医学场景综合评估 |
